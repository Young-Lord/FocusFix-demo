## 主

使用 Electron + NodeJS + [React + Material UI (这东西不太好看)] 创建一个应用，仅实现该应用的 Linux 版本。

该应用主要功能是：每一定时间截取电脑屏幕，基于当前截图使用 openai 服务分析当前用户操作的主题，并与此前用户操作的主题比较，最终生成一个用户专注主题的时间表，以便用户了解自己的时间分配情况。

其中，“主题”由大类和小类组成，比如“学习->编程->Python”、“娱乐->游戏->我的世界”。“主题”最后一项前的内容由用户设定，或在必要时由AI提出新主题。

“与此前用户操作的主题比较”的具体做法：将上次截图时LLM给出的分析，与当前截图LLM给出的分析结合，将二者结合后的内容发送给LLM判断当前主题是否有改变，并由LLM输出一段约100字的分析，以及是否改变主题、新主题是什么。

请根据以上需求，输出你的理解，以及期望的技术栈。

数据库使用SQLite 3。

## 扩展功能 (unused)

基于系统窗口管理器，只获取当前活动窗口的内容、窗口的标题进行分析。
基于浏览器扩展，获取当前网页的内容、标题进行分析，并使用其他工具将网页转为LLM可读的文本。
比较两次截图的图片相似度，若相似度极高，则认为用户操作主题没有变化，不发送给LLM。
提高截图频率，而只以一个较低的频率发送给LLM进行分析，若两次发送给LLM的内容主题不同，则将这两次发送内容之间的截图基于二分法分析，从而找出用户操作主题变化的具体时间点。
局域网/服务器 同步数据
实现 Android, Windows 版本。

此前操作主题：<PLACE1>
可选主题列表：<PLACE2>

当前操作内容概述：xxx
